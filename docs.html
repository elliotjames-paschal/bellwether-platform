Documentation

Everything you need to integrate Bellwether's prediction market data into your application, newsroom, or research pipeline.

Overview

API Live — v1

Bellwether tracks every political prediction market on Polymarket and Kalshi. The Tape API provides canonical tickers, volume-weighted prices, cross-platform matching, and manipulation resistance scores for all 10,700+ markets — updated continuously via live order book and trade data.

Early access. The Tape API is live but access is currently granted by request. Email bellwether@stanford.edu or use the Request Access button to get started. Academic and research use is free.

Base URL

Base URL 
https://api.bellwethermetrics.com/v1

Response Format

All responses are JSON. Timestamps are ISO 8601. Prices are decimal probabilities (0.00 – 1.00). Volume is in contracts traded (not USD).

Authentication

Include your API key in the request header:

Header 
Authorization: Bearer bw_live_xxxxxxxxxxxxxxxx

Keys are issued per organization. The /v1 root and /v1/health endpoints are public; all others require authentication.

Events

Events represent Bellwether's unified view of a market — potentially spanning both platforms. Each event has a slug and an optional canonical ticker.

GET /v1/events List all events

Returns a paginated list of all tracked events. Filter by category or country.

ParameterTypeDescription
categoryoptional string Filter by category. E.g. electoral, monetary_policy, military_security
countryoptional string Filter by country code. E.g. us, ie, fr
limitoptional int Results per page. Default 50, max 200.
offsetoptional int Pagination offset. Default 0.

Response includes total, limit, offset, and markets[] array with slug, ticker, title, k_ticker, pm_token, category, country, platform, total_volume.

GET /v1/events/{slug} Get single event with live data

Returns live price, VWAP, per-platform breakdown, and reportability metrics for an event. Data is fetched from native Polymarket and Kalshi APIs in real time (cached for 30 seconds).

ParameterTypeDescription
slugrequired string Bellwether event slug. E.g. bwr-fitzgerald-win-pres-ie

Response fields

FieldTypeDescription
pricefloatVolume-weighted cross-platform price (VWAP). Falls back to midpoint if insufficient trades.
total_volumeintCombined trade volume across platforms in the VWAP window.
platformsobjectPer-platform data: price, volume, cost_to_move_5c for each of polymarket and kalshi (null if not on that platform).
reportabilitystringrobust, caution, or fragile. Based on cost to move price 5 cents.
cost_to_move_5cintUSD cost to shift price by $0.05 (minimum of both platforms). Null if no order book data.
tierintVWAP data quality tier: 1 (6h window, 10+ trades), 2 (12-24h window), 3 (all available trades).
fetched_atdatetimeWhen upstream data was last fetched. Cached for 30 seconds.
GET /v1/search Search events

Full-text search across market titles, slugs, tickers, and categories. Minimum 2 characters.

ParameterTypeDescription
qrequired string Search query. E.g. california governor 2026

Markets

For direct access to platform-specific data. These endpoints fetch live order book and trade data from the native Polymarket CLOB and Kalshi Elections APIs.

GET /v1/markets/{platform}/{market_id} Platform market data

Returns VWAP, midpoint, order book depth, and reportability for a single platform market.

ParameterTypeDescription
platformrequired string polymarket or kalshi
market_idrequired string Polymarket token ID or Kalshi market ticker

Response fields

FieldTypeDescription
pricefloatBest available price (VWAP if sufficient trades, otherwise midpoint)
vwapfloatVolume-weighted average price. Null if no trades in window.
vwap_window_hoursintWindow used for VWAP: 6, 12, or 24 hours. Null if using all trades.
vwap_trade_countintNumber of trades used in VWAP calculation
vwap_volumeintTotal trade volume in VWAP window
midpointfloatBest bid/ask midpoint from current order book
reportabilitystringrobust, caution, or fragile
cost_to_move_5cintUSD cost to move price $0.05 (minimum of up and down)
tierintData quality tier (1=best, 3=fallback)
GET /v1/markets/combined Cross-platform aggregation

Returns volume-weighted combined data for a matched pair. The combined price is computed per Duffie & Dworczak (2021): weighted by dollar volume across platforms.

ParameterTypeDescription
pm_tokenone required string Polymarket token ID
k_tickerone required string Kalshi market ticker

Provide at least one of pm_token or k_ticker. Provide both for cross-platform aggregation.

Reportability

Every market is scored for manipulation resistance based on live order book depth. The cost_to_move_5c field tells you how many dollars it would take to shift the market price by 5 cents — the core metric behind the reportability tiers.

RatingCost to Move 5¢Guidance
Robust≥ $100,000Safe to cite. Price reflects broad market consensus.
Caution$10,000 – $99,999Cite with context. Note liquidity level.
Fragile< $10,000Do not cite as consensus. Easily manipulated.

Tier 2 data (12-24h VWAP window) downgrades robust to caution and caution to fragile. Tier 3 data (all available trades) is always rated fragile regardless of depth.

Tickers

Bellwether assigns a canonical ticker to every market using an LLM-powered classification system. Tickers follow the format:

Ticker format
BWR-AGENT-ACTION-TARGET-MECHANISM-THRESHOLD-TIMEFRAME

Examples:
BWR-DEM-CONTROL-HOUSE-CERTIFIED-ANY-2026           Party control
BWR-FITZGERALD-WIN-PRES_IE-CERTIFIED-ANY-2025     Election
BWR-FED-CUT-FFR-SPECIFIC_MEETING-5BPS-JAN2026    Monetary policy
BWR-TRUMP-PARDON-MAXWELL-SIGNED_INTO_LAW-ANY-2026 Executive action

Two markets on different platforms match if and only if their tickers are identical. Post-processing rules normalize equivalent resolution mechanisms (e.g. "projected" = "certified" for elections, "primary" = "certified" for nominations).

ComponentDescriptionExamples
AGENTThe actor, party, or institutionDEM, GOP, FED, TRUMP, ZELENSKYY
ACTIONWhat the agent doesWIN, CONTROL, CUT, HIKE, PARDON, LEAVE
TARGETWhat the action applies toPRES, SEN_PA, GOV_CA, FFR, HOUSE
MECHANISMHow the event resolvesCERTIFIED, SPECIFIC_MEETING, ANY_MEANS
THRESHOLDNumeric threshold or qualifierANY, 5BPS, 25BPS, GT_2.0PCT
TIMEFRAMEWhen the event occurs2026, JAN2026, 2025_Q4, DEC2025

Data Glossary

Bellwether provides daily CSV exports of the full dataset. These are the same data that power the API.

political_markets_master.csv
One row per market outcome. All markets classified into 15 political categories.
Identifiers
FieldTypeDescription

pm_condition_idstringPolymarket condition ID (for price lookups)

bellwether_tickerstringCanonical Bellwether ticker for cross-platform matching
Classification
FieldTypeDescription
questionstringFull market question text
political_categorystringOne of 15 categories (see Categories section)
volume_usdfloatTotal trading volume in USD
is_matchedboolTrue if matched with a market on the other platform
price_histories.csv
Daily price observations at multiple time horizons before resolution. One row per market per time horizon.
FieldTypeDescription
market_idstringLinks to master table
days_before_eventintDays before resolution (60, 30, 14, 7, 3, 1)
prediction_pricefloatMarket price (0-1), interpretable as probability
actual_outcomefloatResolution outcome (1.0 = Yes, 0.0 = No)
brier_scorefloatBrier score: (prediction - outcome)²
liquidity_metrics.csv
Order book snapshots and derived liquidity metrics per market.
FieldTypeDescription
market_idstringLinks to master table
spread_meanfloatMean bid-ask spread (absolute, 0-1)
spread_medianfloatMedian bid-ask spread
depth_meanfloatMean total order book depth (contracts at best bid + ask)
bid_depth_meanfloatMean depth on bid side
ask_depth_meanfloatMean depth on ask side
depth_imbalance_meanfloatMean order book imbalance: (bid - ask) / (bid + ask)
n_snapshotsintNumber of order book snapshots collected
